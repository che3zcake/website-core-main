Experiential Scroll Journey — Concept Direction

How should users experience the GitNexus pipeline as they scroll?

Based on GitNexus's actual architecture: processStructure → processParsing → processImports → processHeritage → call-analysis → clustering (communities) → process tracing → search/query

A

Camera Zoom Journey (Recommended)

The particle planet IS the scroll experience. As user scrolls:

  • Hero (top): Full nebula view — "this is your codebase as a living brain"
  • Scroll 20%: Camera zooms in, scattered particles start organizing into clusters — text overlay: "Step 1: Structure — File tree walk"
  • Scroll 40%: Clusters solidify, individual nodes glow — "Step 2: Parsing — Tree-sitter ASTs extract every symbol"
  • Scroll 60%: Edge flows light up between clusters — "Step 3: Resolution — Imports & call chains connect the dots"
  • Scroll 80%: Zoom deep into one cluster, see individual bright nodes — "Step 4: Clustering — Functional communities emerge"
  • Scroll 100%: Pull back to full view, annotations appear — "Your codebase, fully understood"

Pros

  • Most immersive — the product IS the visualization
  • One continuous Three.js canvas, no section breaks
  • Matches the "living brain" metaphor perfectly

Cons

  • Complex to implement (scroll-linked Three.js camera + particle state)
  • Heavy on GPU for the full scroll
  • Mobile might struggle
B

Sticky Canvas + Section Transitions

Particle planet stays fixed/sticky as a background. Each section triggers a state change in the particle visualization:

  • Hero: Full nebula (current state)
  • Section 1: Particles scatter into a file tree pattern
  • Section 2: Particles reform into AST-like branching structures
  • Section 3: Edge flows animate between reformed clusters
  • Section 4: Community clusters pulse and label themselves
  • Section 5: Search ripple — one query lights up a path through the graph

Pros

  • Easier to implement — sections are discrete states
  • Text content scrolls normally over sticky canvas
  • Can gracefully degrade on mobile (static image fallback)

Cons

  • Less cinematic than continuous zoom
  • State transitions need to be smooth
C

Hybrid: Cinematic Hero + Illustrated Sections

Keep the particle planet only in the hero (current). Below it, each pipeline step gets its own mini-visualization:

  • Hero: Full particle nebula with annotations (as-is)
  • Step 1: Animated file tree expanding (CSS/SVG animation)
  • Step 2: Code block morphing into AST nodes (motion library)
  • Step 3: Animated beam diagram showing imports resolving
  • Step 4: Cluster visualization (simplified 2D version of the nebula)
  • Step 5: Terminal showing live query → result animation

Pros

  • Each section is independently polished
  • Lightest on performance
  • Easiest to build incrementally

Cons

  • Least immersive — no continuous visual thread
  • Doesn't leverage the particle planet beyond hero
