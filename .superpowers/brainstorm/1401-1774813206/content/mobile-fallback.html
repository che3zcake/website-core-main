Mobile & Performance — How do we handle it?

The sticky Three.js canvas is GPU-heavy. What happens on phones?

A

Reduced particles + same journey

Keep the sticky canvas on mobile but with 5K particles (vs 20K), no annotations, simpler transitions. Same zig-zag text layout but stacked vertically. The experience is the same, just lighter.

B

Static fallback on mobile

On mobile (<768px), skip the sticky canvas entirely. Show a static hero image/screenshot of the nebula, then the pipeline steps as regular scroll sections with simple fade-in animations. Full experience is desktop-only.

C

Mobile gets the hero only

Mobile sees the particle planet in the hero (reduced to 5K particles, no sticky). Below that, pipeline steps render as normal cards with icons — similar to the current How It Works but with step numbers and descriptions. Desktop gets the full cinematic experience.
